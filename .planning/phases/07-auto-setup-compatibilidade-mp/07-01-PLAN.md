# Phase 7 Plan 01: Auto-Setup + Compatibilidade MP

## Objective

- **What:** Implementar auto-geração do `config.ini` quando ausente, melhorar diagnóstico de periféricos (com mensagens acionáveis) e endurecer o discovery para ambiente multiplayer.
- **Why:** Reduzir atrito de instalação e tornar falhas previsíveis/explicáveis (sem depender de estado local do jogador).
- **Output:** Bootstrap que cria config e loga defaults, periféricos com diagnóstico útil, e testes cobrindo os fluxos críticos.

## Threat Model (ASVS L1)

- **T-07-01 (Alta):** Sobrescrever `config.ini` do usuário ao regenerar defaults → mitigação: escrever apenas quando o arquivo não existir; nunca reescrever automaticamente.
- **T-07-02 (Média):** Criar arquivo de mapeamentos inválido e quebrar equivalências/tiers → mitigação: gerar skeleton com campos esperados e validar decode/estrutura mínima.
- **T-07-03 (Média):** Crash/loop de spam em MP por falhas de permissão/periféricos → mitigação: `pcall` no discovery, mensagens acionáveis e degradação previsível.

## Tasks

<task type="code">
  <name>Auto-gerar `config.ini` com template padrão e logar defaults aplicados</name>
  <files>lib/config.lua, lib/bootstrap.lua</files>
  <action>
    Em lib/config.lua:
    (1) Adicionar uma constante `DEFAULT_INI` (string multi-linha) com o conteúdo padrão do `config.ini` que o projeto já usa.
    (2) Adicionar função `Config.ensureDefaults(path)` que:
        - Se `fs.exists(path)` for true: retorna `{ created=false }`.
        - Se não existir: escreve `DEFAULT_INI` em `path` (modo "w") e retorna `{ created=true, defaults=parseIni(DEFAULT_INI) }`.
        - Se ocorrer erro ao escrever: retorna `{ created=false, err=tostring(err) }` (sem lançar erro).
    Em lib/bootstrap.lua:
    (3) Antes de `Config.load("config.ini")`, chamar `local ensure = Config.ensureDefaults("config.ini")`.
    (4) Depois de criar `logger`, se `ensure.created` for true:
        - `logger:info("config.ini criado com defaults")`
        - Para cada seção/chave em `ensure.defaults`, logar uma linha (ex.: `logger:info("Default aplicado", { section="core", key="log_level", value="INFO" })`).
    (5) Garantir que o fluxo continua mesmo se `ensure.err` existir (logar um ERROR e seguir para `Config.load`).
  </action>
  <verify>
    - Em lib/config.lua existe `DEFAULT_INI` e `function Config.ensureDefaults` (grep por "DEFAULT_INI" e "ensureDefaults").
    - Em lib/bootstrap.lua há chamada a `Config.ensureDefaults("config.ini")` antes de `Config.load` (grep por "ensureDefaults" antes de "Config.load").
  </verify>
  <done>
    `config.ini` é criado automaticamente quando ausente, e o log registra explicitamente que defaults foram aplicados.
  </done>
</task>

<task type="code">
  <name>Garantir `data/mappings.json` com estrutura mínima (CFG-03) quando ausente</name>
  <files>modules/equivalence.lua</files>
  <action>
    Em modules/equivalence.lua:
    (1) Antes de ler o DB, se `fs.exists(DB_PATH)` for false, criar o arquivo com JSON skeleton:
        - `{ "items": {}, "classes": {}, "tier_overrides": {}, "gating": { "by_building_type": {} } }`
    (2) A escrita deve ser feita via `Util.writeFile(DB_PATH, Util.jsonEncode(tbl))`.
    (3) Se a escrita falhar, não lançar erro; deixar `loadDb()` retornar a estrutura mínima em memória.
    (4) Quando `state.logger` existir, logar uma vez: "data/mappings.json ausente; arquivo criado com estrutura mínima" (INFO).
  </action>
  <verify>
    - modules/equivalence.lua contém a estrutura mínima (grep por "by_building_type" e "tier_overrides").
    - Ausência do arquivo não causa erro de `jsonDecode` (loadDb retorna tabelas mesmo com arquivo ausente).
  </verify>
  <done>
    O sistema nunca inicia com DB de mapeamentos "nil/vazio inesperado" por falta do arquivo; existe um arquivo base editável.
  </done>
</task>

<task type="code">
  <name>Melhorar diagnóstico de periféricos e endurecer discovery para multiplayer</name>
  <files>modules/peripherals.lua</files>

  </verify>
  <done>

  </done>
</task>
