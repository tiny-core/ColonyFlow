local function jobSymbol(jobState)

  if s == "done" then return "OK" end
  if s:find("wait") or s:find("retry") or s:find("await") then return "AG" end

end

<threat_model>
## Trust Boundaries

| Boundary | Descrição |
|----------|-----------|
| work[id] → pre-pass | Dados de work[id] lidos sem validação de tipo no sort |
| retryEligible → _processRequest | Lista construída por iteração sobre requests; r.id pode ser qualquer tipo |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-21-01 | Denial of Service | Engine:tick() pre-pass | mitigate | Pre-pass respeita `rqLimit` via `if processed >= rqLimit then break end` — se todos os requests forem waiting_retry elegíveis, o pre-pass não pode consumir mais que `requests_per_tick` por tick; loop normal simplesmente não roda naquele tick (D-05, by design) |
| T-21-02 | Denial of Service | retry_count overflow | accept | Lua numbers são doubles (2^53 precisão inteira); retry_count incrementado 1x por tentativa; não persiste — zera no reboot. Em operação contínua típica (horas/dias) o valor nunca aproxima de overflow; sem risco real |
| T-21-03 | Information Disclosure | Badge [R:N] no monitor | accept | retry_count é contador operacional sem PII; exibição em monitor local do CC:Tweaked; sem risco de disclosure |
| T-21-04 | Denial of Service | Infinite retry loop (requests nunca saindo de waiting_retry) | mitigate | Pre-pass não altera status do request — _processRequest segue o fluxo normal (pode resolver ou re-colocar em waiting_retry com novo next_retry). O backoff de next_retry existente (work.next_retry = ctx.nowEpoch + N) garante que cada tentativa tem janela de espera; requests não ficam processando 100% do budget em cada tick sem backoff |
| T-21-05 | Tampering | craft.started_at nil/não-numérico no sort | mitigate | Fallback `math.huge` aplicado via `(wa and type(wa.craft) == "table" and tonumber(wa.craft.started_at)) or math.huge` — requests sem craft chegam por último, sem panic |
</threat_model>
