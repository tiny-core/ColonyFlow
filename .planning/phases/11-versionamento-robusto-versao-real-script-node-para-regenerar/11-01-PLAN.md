# Phase 11 Plan 01: Versao real + gerador de manifesto

## Objective

- **What:** Introduzir versao real (SemVer) no `manifest.json`, persistir em `data/version.json` via instalador e adicionar um script Node para regenerar `manifest.json` com lista ordenada e `size`.
- **Why:** Permitir update check confiavel (fase 12), auditoria de versao instalada e reduzir risco de downloads parciais (integridade por `size`).
- **Output:** `VERSION` + schema atualizado do manifesto + ajustes no instalador + `tools/gen_manifest.js` + tests + checklist de verificacao.

## Threat Model (ASVS L1)

- **T-11-01 (Alta):** Atualizacao aplica manifesto malformado e grava `data/version.json` inconsistente, quebrando update-check futuro e dificultando recovery -> mitigacao: validacao estrita do campo `version` e escrita consistente (sempre gravar `version` apenas quando manifesto validar).
- **T-11-02 (Media):** Script de manifesto inclui arquivos errados (ex.: `.planning/` ou `data/version.json`) e o instalador passa a sobrescrever dados do usuario -> mitigacao: allowlist por pastas/extensoes + denylist explicita (data/version.json, data/install.json, logs, backups).
- **T-11-03 (Media):** Versao no manifesto diverge do repo (operador acha que esta em X.Y.Z mas binarios sao outros) -> mitigacao: `VERSION` como fonte canonica + gerador sempre copia esse valor para `manifest.json`.
</threat_model>

    - Nao existem strings com acentos nas novas mensagens do instalador (verificacao manual por leitura do diff).

  <files>lib/version.lua, tests/run.lua</files>
  <read_first>
    - lib/util.lua
    - tests/run.lua
    - tools/install.lua
  </read_first>
  <action>
    Criar `lib/version.lua` com funcoes puras:
    - `Version.isValid(semver)`:
      - Retorna `true/false` para strings no formato `X.Y.Z` (somente numeros).
    - `Version.compare(a, b)`:
      - Retorna `-1` se `a < b`, `0` se iguais, `1` se `a > b`.
      - Comparacao numerica por componente (major, minor, patch).
    - `Version.readInstalled()`:
      - Le `data/version.json` (quando existir) e retorna `{ version=..., ref=..., manifest_url=... }` ou `nil` se ausente/invalido.

    Em `tests/run.lua`, adicionar testes:
    - `semver_is_valid` cobrindo casos validos (`0.0.0`, `1.2.3`) e invalidos (`1.2`, `1.2.3.4`, `a.b.c`, `01.2.3` se for considerado invalido).
    - `semver_compare` cobrindo ordenacao (`1.0.0` > `0.9.9`, `1.2.0` < `1.2.1`, igualdade).
  </action>
  <acceptance_criteria>
    - Existe `lib/version.lua` e contem `isValid` e `compare` (grep).
    - `tests/run.lua` contem testes `semver_is_valid` e `semver_compare` (grep).
    - `startup test` passa (manual in-world).
  </acceptance_criteria>
  <verify>

  </verify>

      - `.planning/**`, `.trae/**`, `logs/**`, `data/install.json`, `data/version.json`, `data/backups/**`, `data/.install_tmp/**`
    - Escreve `manifest.json` (pretty) no root.
  </action>

    - Rodar `node tools/gen_manifest.js` reescreve `manifest.json` contendo `version`, `generated_utc` e `files` ordenado (verificacao manual).
    - `manifest.json` contem `size` como numero em pelo menos 3 entradas (grep por `\"size\": [0-9]+`).
    - `manifest.json` marca `config.ini` e `data/mappings.json` com `\"preserve\": true` (grep).
  </acceptance_criteria>
