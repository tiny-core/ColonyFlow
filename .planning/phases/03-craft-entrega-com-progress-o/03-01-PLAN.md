# Phase 3 Plan 01: Craft + Entrega com Progressão

## Objective

- **What:** Integrar o ME Bridge para (1) consultar estoque/craftabilidade, (2) iniciar craft apenas do faltante, (3) exportar itens para o destino padrão com validação pós-entrega, (4) impedir duplicidade de jobs e (5) aplicar tier gating por building/worker.
- **Why:** Esta é a fase que fecha o ciclo de valor: pedido → decisão → craft → entrega, sem quebrar progressão e sem gerar spam de jobs.
- **Output:** Engine com estados `crafting`/`delivering`/`waiting_retry`/`blocked_by_tier`, ME wrapper compatível com variações de API, gating por building e testes cobrindo as regras críticas.

## Threat Model (ASVS L1)

- **T-03-01 (Alta):** Loop de spam (craft/export) por falhas transitórias → mitigação: lock TTL + backoff + estados explícitos.
- **T-03-02 (Média):** Entrega incorreta por corrida de inventário → mitigação: validação pós-entrega por snapshot e tolerância a concorrência.
- **T-03-03 (Média):** Pular progressão por gating falho → mitigação: fail closed quando building não resolver + filtro por tier permitido + fallback para tier menor.
</threat_model>
