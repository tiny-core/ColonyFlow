# Phase 10 Plan 01: Config CLI (config.ini + perifericos)

## Objective

- **What:** Implementar `startup config` para editar `config.ini` com foco em `[peripherals]`, `[core]` (inclui logs) e `[delivery]`, com validacao bloqueante e salvamento seguro preservando comentarios/ordem.
- **Why:** Reduzir atrito operacional e evitar falhas por configuracao incorreta de perifericos/parametros.
- **Output:** `modules/config_cli.lua` + patcher/writer de INI + testes + guia de verificacao.

## Threat Model (ASVS L1)

- **T-10-01 (Alta):** Corromper `config.ini` ao salvar (arquivo truncado/estado parcial) -> mitigacao: backup sempre + escrita atomica (`.tmp` + `fs.move`) e falha bloqueante se nao conseguir escrever.
- **T-10-02 (Media):** Destruir comentarios/ordem e chaves desconhecidas (perda de documentacao/ajustes manuais) -> mitigacao: patch in-place por linhas, sem re-serializar do parse.
- **T-10-03 (Media):** Configurar perifericos incorretos e causar downtime (UI/engine nao sobem) -> mitigacao: validar `isPresent+wrap` antes de salvar e impedir monitores duplicados.
</threat_model>

<task type="code">
  <name>Implementar patcher/writer de INI com preservacao + escrita segura</name>
  <files>lib/config.lua, lib/util.lua</files>
  <read_first>
    - lib/config.lua
    - lib/util.lua
    - config.ini
  </read_first>
  <action>
    Em `lib/config.lua`, adicionar funcoes novas (sem quebrar chamadas existentes):
    - `Config.patchIniText(linesOrText, updatesBySection)`:
      - Entrada: texto (string) ou array de linhas; `updatesBySection` no formato `{ [section] = { [key] = valueString } }`.
      - Saida: `{ text = <novo_texto>, changes = { { section, key, old, new, op }... } }`.
      - Regras:
        - Preservar linhas que nao sao `key=value` dentro da secao (comentarios e linhas desconhecidas).
        - Atualizar apenas o valor de `key=` quando a chave existe na secao.
        - Inserir chaves ausentes no fim da secao (antes da proxima secao).
        - Se a secao nao existir, criar no fim do arquivo com header `[section]` e chaves.
    - `Config.patchIniFileAtomic(path, updatesBySection, opts)`:
      - Criar backup sempre em `opts.backup_dir` (default `data/backups`) com timestamp UTC.
      - Escrever em `path .. ".tmp"` e trocar para `path` via `fs.move`.
      - Retornar `{ ok=true, changes=[...], backup_path=... }` ou `{ ok=false, err="..." }`.

    Em `lib/util.lua`, adicionar helpers pequenos para suportar isso:
    - `Util.copyFile(src, dst)` (readFile + writeFile).
    - `Util.writeFileAtomic(path, content)` (tmp + move).
    - `Util.isoTimestampUtc()` (string `YYYY-MM-DDTHHMMSSZ` baseada em `os.date("!")`).
  </action>
  <acceptance_criteria>
    - `lib/config.lua` contem `patchIniText` e `patchIniFileAtomic` (grep).
    - `lib/util.lua` contem `writeFileAtomic` e `isoTimestampUtc` (grep).
    - `patchIniText` nao faz re-serializacao completa a partir de `parseIni` (verificacao manual por leitura do codigo: opera por linhas).
  </acceptance_criteria>
  <verify>
    - `tests/run.lua` (apos task de testes) valida preservacao de comentarios e insercao de chaves.
  </verify>
  <done>
    Existe um mecanismo de salvar `config.ini` que preserva o texto e reduz risco de corrupcao.
  </done>
</task>
