Implement the data layer for stuck-request alerts: set and clear `stuck_since_ms` on `work[id]` at every status transition in engine.lua, copy it through the snapshot contract in snapshot.lua, and add the `alert_stuck_minutes=5` default to config.lua.

Purpose: The UI (Plan 02) reads `stuck_since_ms` from the snapshot to compute elapsed time and render alerts. This plan produces the complete data contract the UI depends on.
Output: Three files modified — engine.lua (lifecycle), snapshot.lua (copy), config.lua (default).
</objective>
