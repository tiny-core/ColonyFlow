# Plan 04-01: Implementar UI dual-monitor e ferramentas operacionais

<objective>
Tornar o sistema operável em produção com dois monitores atualizados sem flicker (diff-based rendering), suportando paginação via toque. Adicionar um editor TUI interativo para mapeamentos com hot-reload no motor, e garantir logs claros de substituições.
</objective>

<threat_model>
- **Data Loss in Editor**: Editor must validate JSON before saving. Backup previous JSON if necessary.
- **UI Freeze**: UI rendering must yield or be fast enough not to trigger "Too long without yielding" error.
- **Event Loop Block**: TUI operations must block only the terminal, not the background engine, or we run them in parallel using `parallel.waitForAny`.
</threat_model>

<tasks>

<task>
<description>Implement line-based double buffering in UI</description>
<read_first>
- components/ui.lua
</read_first>
<action>
1. In `components/ui.lua`, add a `self.buffers = { requests = {}, status = {} }` table to `UI.new()`.
2. Create a helper `function UI:drawText(deviceKey, mon, x, y, text, fg, bg)` that compares `text` with `self.buffers[deviceKey][y]`.
3. If different, call `mon.setCursorPos(x, y)`, `mon.setTextColor(fg)`, `mon.setBackgroundColor(bg)`, `mon.write(text)`, and update the buffer.
4. Remove `term.clear()` and `term.redirect()` usage from `renderRequests` and `renderStatus`. Instead, pass `self` and `deviceKey` so they can use `self:drawText`.
5. Clear the screen only on initialization or monitor resize.
</action>
<acceptance_criteria>
- `components/ui.lua` contains `self.buffers` initialization.
- `components/ui.lua` contains a method `drawText` that compares strings and only writes if changed.
- `term.clear()` is removed from the `tick()` rendering loop.
</acceptance_criteria>
</task>

<task>
<description>Implement pagination and touch events for Monitor 1</description>
<read_first>
- components/ui.lua
- startup.lua
</read_first>
<action>
1. In `components/ui.lua`, add `UI:handleEvent(event, side, x, y)` to handle `monitor_touch` events.
2. In `UI:handleEvent`, if `side` matches `state.devices.monitorRequests` name, adjust `self.page` based on `x` (e.g., left half = prev page, right half = next page) and reset an `autoRotationTimer`.
3. In `renderRequests`, calculate `pageSize` based on monitor height `h`. Use `self.page` instead of `os.epoch` math. If `autoRotationTimer` expires, increment `self.page` automatically.
4. Mark substitutions clearly: compare `r.items[1].name` with `job.chosen`. If different, append `(S)` to the displayed item name.
5. In `startup.lua`, update the main loop to use `os.startTimer` and `os.pullEventRaw`. Pass relevant events to `ui:handleEvent`.
</action>
<acceptance_criteria>
- `components/ui.lua` contains `UI:handleEvent(event, side, x, y)`.
- `components/ui.lua` calculates `pageSize` dynamically based on monitor height.
- Substitutions are marked with `(S)` in `renderRequests`.
- `startup.lua` uses `os.pullEventRaw` and forwards events to `ui:handleEvent`.
</acceptance_criteria>
</task>
