# Phase 13 Plan 01: Operabilidade do update-check (config + backoff + detalhes)

## Objective

- **What:** Adicionar configuracao de update-check via `config.ini`, backoff/retry em falhas e uma tela de detalhes (status/erro/url) no Monitor 2.
- **Why:** Tornar o update-check previsivel em MP/servidores com HTTP bloqueado e facilitar diagnostico sem abrir terminal.
- **Output:** Config + update_check melhorado + UI de detalhes + testes + checklist.
