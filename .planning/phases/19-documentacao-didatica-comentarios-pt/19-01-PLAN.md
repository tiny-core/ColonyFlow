# Phase 19 Plan 01: Documentacao Didatica + Comentarios (PT)

## Objective

- **What:** Criar documentacao "publica" em `docs/` (guia + 3 docs de apoio), linkar via `README.md`, e adicionar comentarios curtos (por que/invariantes) nos modulos-chave.
- **Why:** Facilitar leitura do codigo e manutencao sem renomear APIs/identificadores.
- **Output:** `docs/LEIA-ME-DO-CODIGO.md`, `docs/{SUMMARY,ARCHITECTURE,PITFALLS}.md`, README com secao "Documentacao", comentarios nos arquivos-chave, e checklist de verificacao.

<threat_model>
## Threat Model

### Assets
- Clareza para novos leitores (guia encontra-se facilmente)
- Manutencao (comentarios ajudam sem virar ruido)
- Estabilidade (comentarios nao mudam runtime; testes continuam verdes)

### Threats (block on: high)
- **T-19-01 (high):** Documentacao morta (docs nao linkados / escondidos) -> mitigacao: README com secao "Documentacao" e links diretos para docs/.
- **T-19-02 (high):** Comentarios alteram comportamento por edicao acidental -> mitigacao: mudancas apenas em comentarios; rodar `startup test` apos cada arquivo Lua tocado.
- **T-19-03 (high):** Comentarios viram ruido (repetem codigo) -> mitigacao: padrao estrito: apenas por que/invariantes/contratos; sem comentar linhas obvias.
</threat_model>
